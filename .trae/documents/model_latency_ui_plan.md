# 模型延迟测试 UI 优化计划

## 任务概述
优化模型延迟测试功能的用户体验：
1. 将"模型延迟测试"按钮从侧边栏菜单移到 AI 模型选择区域下方
2. 添加实时终端式日志输出，让用户看到测试进度而不是干等

## 当前问题
- 模型延迟测试按钮在侧边栏菜单中，位置不够合理
- 测试过程中用户只能看到"基准测试中..."，没有实时进度反馈

## 修改方案

### 1. 移动按钮位置
**目标位置：** AI 模型选择框下方

**需要修改的文件：**
- Sidebar/index.tsx
  - 删除当前的"⚡ 模型延迟测试"按钮（从菜单列表中移除）
  - 在 AI 模型选择框下方添加该按钮

### 2. 添加实时日志输出
**目标：** 像终端一样实时显示测试进度

**需要修改的文件：**
- ModelLatencyTest/index.tsx
  - 添加日志状态管理
  - 添加终端式日志显示区域
  - 修改测试函数，在关键节点添加日志输出

**日志内容示例：**
```
[10:23:45] 🚀 开始测试模型延迟...
[10:23:45] ⏳ 正在测试 qwen3-max...
[10:23:48] ✅ qwen3-max: 首字节 234ms, 总耗时 456ms
[10:23:48] ⏳ 正在测试 glm-4.7...
...
[10:24:12] 📊 测试完成！
```

## 实施步骤

1. 修改 Sidebar/index.tsx
   - 从菜单按钮列表中移除模型延迟测试按钮
   - 在 AI 模型选择区域下方添加该按钮

2. 修改 ModelLatencyTest/index.tsx
   - 添加 logs 状态数组
   - 添加 addLog 函数，自动添加时间戳
   - 添加终端样式日志显示组件
   - 修改 testAllModelsLatency 和 runLatencyBenchmark 函数，在关键步骤调用 addLog

3. 可选：添加日志自动滚动到底部功能

## UI 设计

### 按钮样式
- 位置：AI 模型选择框下方
- 样式：与周围元素协调，使用紫色主题色
- 文字："⚡ 模型延迟测试"

### 日志终端样式
- 背景：深色（dark:bg-gray-900）
- 字体：等宽字体（font-mono）
- 文字颜色：绿色（text-green-400）表示成功，红色表示错误
- 高度：固定高度，可滚动
- 自动滚动：新日志自动滚动到底部

## 预期结果
- 模型延迟测试按钮位于 AI 模型选择区域下方
- 测试过程中实时显示日志输出
- 用户可以清楚看到每个模型的测试进度和结果
